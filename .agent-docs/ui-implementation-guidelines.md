# UI 实施规范（会话沉淀）

更新日期：2026-04-13

## 1. 总体风格
- 风格目标：简洁、节约、偏 Codex Agent 风格。
- 禁止事项：为信息再额外套无必要容器，避免“盒子套盒子”的视觉堆砌。
- 模块表达：用清晰标题和最少层级表达，不用冗余说明性文案。

## 2. 结构与层级
- 右侧主区只保留一级主模块（FoxCode）和必要分区（额度、状态）。
- 同一分区内优先单层内容布局，避免重复边框与重复背景块。
- 信息层级顺序：主值 > 状态/标签 > 辅助说明。

## 3. 状态可视化
- 状态条应尽量横向铺满可用宽度，不出现明显短条留白。
- 可视化色彩语义固定：
  - `up`：绿色
  - `down`：红色
  - `unknown`：中性灰
- 可视化为辅助信息，不应喧宾夺主；文本信息仍需完整可读。

## 4. 文案与命名
- 不出现“总模块/子模块”字样。
- 文案短句化，优先名词性标题（如“额度”“状态”）。
- 面向终端用户的文案要可扫描，不写实现细节。

## 5. 交互原则
- 所有关键交互必须有反馈（loading、hover、成功/失败提示）。
- hover 提示必须在可视区域内，不能遮挡关键信息。
- 按钮与状态点的 hover 态应明显但克制。
