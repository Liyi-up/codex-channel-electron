## 1. 文档治理
- 目录用途：沉淀可复用的实现约束与复杂模块说明，减少后续会话反复读代码。
- 适用场景：UI 风格改造、状态模块改动、交互回归检查。
- 维护规则：修改实现时优先同步对应文档，再提交代码。
