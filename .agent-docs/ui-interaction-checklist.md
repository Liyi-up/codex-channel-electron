# UI 交互验收清单

更新日期：2026-04-13

## 1. 视觉检查
- 是否仍存在无必要内层卡片/边框叠加。
- 状态条是否基本铺满行宽，左右留白是否合理。
- 颜色语义是否一致（绿/红/灰）。

## 2. hover 与 popover
- hover 到状态点时是否出现提示。
- popover 在左侧点、居中点、右侧点是否都不越界。
- popover 是否遮挡关键操作按钮或标题。

## 3. loading 与反馈
- 获取额度按钮是否有 loading 文案与禁用态。
- 状态刷新过程中是否有可见反馈。
- 历史刷新点击后是否有转圈与结果提示。

## 4. 信息完整性
- 额度区是否展示：按量额度、月卡额度、账号、更新时间。
- 状态区是否展示：可用率、线路名、时间窗口、相对时间、最近检测时间。
- 错误场景是否有降级文案（接口异常/数据暂不可用）。

## 5. 提交前动作
- 执行 `npm run build`。
- 手动检查深浅主题下可读性。
- 至少验证一次 hover、一次加载、一次错误提示路径。
